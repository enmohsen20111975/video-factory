# 📗 النظام المرجعي الكامل - Content Extractor Pipeline
## نظام استخراج المحتوى من الكتب المدرسية المصرية وتحويله إلى فيديوهات تعليمية

---

**آخر تحديث:** يونيو 2026  
**الإصدار:** 1.0.0  
**المنصة المستهدفة:** Windows 11, NVIDIA GPU 8GB VRAM, 32GB RAM  
**اللغات:** Python 3.10+ (Backend), TypeScript/React (Frontend - Remotion)  
**الهدف:** تحويل كتب PDF المدرسية (المعاصر، الامتحان، التفوق) إلى محتوى فيديو تعليمي تلقائياً

---

# 🏗️ الفصل الأول: نظرة عامة على النظام (System Overview)

## 1.1 ما هذا النظام؟

هذا النظام هو **خط أنابيب (Pipeline)** كامل يقوم بأتمتة تحويل الكتب المدرسية PDF إلى فيديوهات تعليمية. يعمل محلياً بالكامل على اللابتوب دون الحاجة لخدمات سحابية.

```
📚 كتاب PDF (المعاصر / الامتحان / التفوق)
    ↓
┌─────────────────────────────────────────────┐
│ Content Extractor Pipeline  (4 مراحل)       │
│ المرحلة 1: PDF → صور (pdf-to-images.py)     │
│ المرحلة 2: صور → JSON (extract-page.py)    │
│ المرحلة 3: JSON ← دروس (merge-pages.py)    │
│ المرحلة 4: دروس ← Markdown+JSON (generate) │
└─────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────┐
│ Video Factory Pipeline   (5 خطوات)          │
│ run-factory.js → TTS → Remotion → FFmpeg   │
└─────────────────────────────────────────────┘
    ↓
🎬 فيديو تعليمي جاهز (public/lesson-XX.mp4)
```

## 1.2 لماذا هذا النظام؟
- **المشكلة الأصلية:** المستخدم كان يحاول معالجة كتب كاملة دفعة واحدة مما يتسبب في انهيار النظام (OOM)
- **الحل:** معالجة صفحة واحدة كل مرة مع فترة تبريد إجبارية بين كل صفحة وأخرى
- **الميزة الرئيسية:** يعمل محلياً بالكامل (بدون إنترنت، بدون APIs سحابية)
- **المرونة:** يدعم أي كتاب PDF مع تعديل ملف book-index.json

**المسار:** `config/extraction-prompt.txt`

| `page_type` | string | lesson, exercise, example, summary, empty, unclear |
| `lesson_title` | string? | عنوان الدرس |
| `content_summary` | string | ملخص الصفحة (2-3 جمل) |
| `definitions` | array | المصطلحات مع تعريفاتها |

| `exercises` | array | التمارين |

# 🎬 الفصل الخامس: Video Factory Pipeline (إنتاج الفيديو)

## 5.1 run-factory.js — مشغل مصنع الفيديو
**المسار:** `video-factory/run-factory.js`  
**عدد الأسطر:** 103

### سير العمل (5 خطوات):
```
① تحميل بيانات الدرس (src/data/lesson-XX.json)
② توليد الصوت (generate_tts.py → MP3 + Timestamps)
③ عرض الفيديو (npx remotion render → MP4 خام)
④ ضغط الفيديو (FFmpeg → H.264 MP4)
⑤ تنظيف الملفات المؤقتة
```

### شرح كل خطوة:

**الخطوة 0: تحليل المعاملات**
```javascript
let lessonName = "ohm-law";  // الاسم الافتراضي
// node run-factory.js --lesson=lesson-02
```

**الخطوة 1: التحقق من البيانات**
```javascript
const dataPath = `src/data/${lessonName}.json`;
if (!fs.existsSync(dataPath)) process.exit(1);
```

**الخطوة 2: توليد الصوت**
```javascript
execSync(`python "scripts/generate_tts.py" \
  --text "${voiceoverText}" \
  --output-audio "public/voiceovers/${lessonName}.mp3" \
  --output-timestamps "public/timestamps/${lessonName}.json"`);
```
- **المدخل:** `voiceoverText` من JSON
- **المخرج 1:** `public/voiceovers/lesson-02.mp3`
- **المخرج 2:** `public/timestamps/lesson-02.json`
- **التقنية:** edge-tts (Microsoft Edge TTS, صوت `ar-EG-SalmaNeural`)

**الخطوة 3: عرض الفيديو (Remotion)**
```javascript
execSync(`npx remotion render LessonVideo "${rawVideoPath}" \
  --props="${dataPath}" --concurrency=4`);
```
- المكون: `LessonVideo.tsx` (7 مشاهد, 75 ثانية)
- المدة: 2-5 دقائق للعرض

**الخطوة 4: ضغط الفيديو (FFmpeg)**
```javascript
execSync(`ffmpeg -i "${rawVideoPath}" \
  -vcodec libx264 -crf 22 -preset fast \
  -pix_fmt yuv420p -acodec aac -b:a 128k "${finalVideoPath}"`);
```
- الترميز: H.264 (متوافق مع كل المتصفحات)
- الجودة: CRF 22

**الخطوة 5: التنظيف**
```javascript
fs.unlinkSync(rawVideoPath);  // حذف الفيديو الخام
```

# 🚨 الفصل السابع: استكشاف الأخطاء (Troubleshooting)

## 7.1 "Ollama not found"

**الرسالة:** `❌ ollama package not installed` أو `Error: model not available`

**الأسباب:**
- Ollama غير مثبت
- Ollama غير شغال
- النموذج غير محمل

**الحل:**
```bash
# تأكد من أن Ollama شغال
ollama serve

# تأكد من تحميل النماذج
ollama list

# إذا النموذج مش موجود
ollama pull qwen2-vl:7b
```

## 7.2 "GPU VRAM too high"

**الرسالة:** `⚠️ GPU VRAM high: 7500MB/7168MB`

**الأسباب:**
- برامج أخرى تستخدم GPU (ألعاب، متصفحات)
- عملية Ollama سابقة ما زالت في الذاكرة
- جاري تشغيل أكثر من نموذج Ollama معاً

**الحل:**
```bash
# شوف مين بيستخدم GPU
nvidia-smi

# أوقف العمليات الزائدة
# استخدم مدير المهام (Task Manager)
# أو اعمل إعادة تشغيل Ollama
ollama stop --all
ollama serve
```

## 7.3 OOM (Out of Memory)

**الرسالة:** `❌ OOM error - model qwen2-vl:7b ran out of memory`

**الأسباب:**
- VRAM غير كافية (أقل من 4GB)
- الصور أكبر من 512px
- هل هناك أكثر من صورة في الطلب؟

**الحل:**
- استخدم `--model gemma3:4b` أو `--model qwen2-vl:2b`
- تأكد من أن الصور 512px كحد أقصى
- أغلق كل تطبيقات GPU الأخرى

## 7.4 انهيار السكربت في منتصف الاستخراج

**الرسالة:** أي خطأ أثناء `extract-page.py`

**الحل:**
```
لا تفعل شيئاً! فقط أعد تشغيل نفس الأمر.
السكربت سيكتشف الصفحات 1-44 موجودة ويبدأ من 45.
```

## 7.5 فشل تفسير JSON

**الرسالة:** `❌ Page 45 Failed: Could not parse JSON from VLM response`

**السبب:** نموذج VLM لم يخرج JSON صحيح (قد يكون خرج نصاً فقط)

**الحل:**
1. افحص `raw-json/page_0045.json` وشوف `raw_response`
2. إذا المشكلة متكررة → استخدم نموذج أفضل أو قلل `temperature`
3. استخدم `--force` لإعادة محاولة الصفحات الفاشلة

## 7.6 صفحات كثيرة "فاشلة" أو "غير واضحة"

**الرسالة:** `❌ Failed: 45` من أصل 200

**الأسباب:**
- جودة الصورة رديئة (مسح ضوئي سيئ)
- النص صغير جداً أو مشوه
- النموذج غير مناسب

**الحل:**
- جرب DPI أعلى: `--dpi 200`
- استخدم النموذج الأكبر: `qwen2-vl:7b`
- تأكد من أن الصفحة نظيفة ومقروءة

## 7.7 النتائج باللغة الإنجليزية وليس العربية

**المشكلة:** VLM يخرج النص باللغة الإنجليزية

**السبب:** الـ Prompt يحتوي على تعليمات باللغة الإنجليزية

**الحل:** هذه مشكلة معروفة في النماذج متعددة اللغات. حاول:
1. تعديل `extraction-prompt.txt` ليكون بالعربية أكثر
2. أو أضف "تأكد من أن النص المستخرج بالعربية الفصحى" في بداية الـ Prompt

## 8.2 run-factory.js — مشغل مصنع الفيديو

**المسار:** `video-factory/run-factory.js`  
**الوظيفة:** تشغيل مصنع الفيديو بالكامل: نص ← صوت ← فيديو ← ضغط  
**اللغة:** JavaScript (Node.js)  
**عدد الأسطر:** 103 سطراً

### سير العمل (5 خطوات)

```
الخطوة 1: تحميل بيانات الدرس من src/data/lesson-XX.json
الخطوة 2: توليد الصوت (voiceover MP3) والـ timestamps JSON
الخطوة 3: عرض الفيديو عبر Remotion CLI (npx remotion render)
الخطوة 4: ضغط الفيديو الخام عبر FFmpeg (H.264)
الخطوة 5: حذف الملفات المؤقتة
```

### شرح كل خطوة بالتفاصيل

#### الخطوة 0: تحليل المعاملات

```javascript
// run-factory.js - الأسطر 1-15
const args = process.argv.slice(2);
let lessonName = "ohm-law";  // الاسم الافتراضي

args.forEach(arg => {
  if (arg.startsWith("--lesson=")) {
    lessonName = arg.split("=")[1];
  }
});
```

- المعامل `--lesson` يحدد اسم الدرس (بدون امتداد .json)
- الاسم الافتراضي هو `ohm-law` (للتوافق مع النظام القديم)
- أمثلة: `node run-factory.js --lesson=lesson-02`

#### الخطوة 1: التحقق من وجود ملف البيانات

```javascript
// run-factory.js - الأسطر 19-27
const dataPath = path.join(__dirname, `src/data/${lessonName}.json`);
if (!fs.existsSync(dataPath)) {
  console.error(`❌ Error: Data file not found at ${dataPath}`);
  process.exit(1);
}
```

- يبحث عن ملف JSON في `video-factory/src/data/`
- إذا الملف مش موجود → يطبع خطأ ويخرج
- **ملاحظة:** هذا هو المكان الذي تضع فيه مخرجات `generate-markdown.py` ملفاتها

#### الخطوة 2: توليد الصوت والـ Timestamps

```javascript
// run-factory.js - الأسطر 39-57
const scriptText = lessonData.introText || lessonData.voiceoverText;
if (scriptText) {
  console.log("🎙️ Step 1: Generating voiceover and timestamps...");
  
  const pythonScriptPath = path.join(__dirname, "scripts/generate_tts.py");
  const audioOutPath = path.join(__dirname, `public/voiceovers/${lessonName}.mp3`);
  const timestampsOutPath = path.join(__dirname, `public/timestamps/${lessonName}.json`);
  
  execSync(`python "${pythonScriptPath}" --text "${scriptText}" 
    --output-audio "${audioOutPath}" --output-timestamps "${timestampsOutPath}"`, 
    { stdio: "inherit" });
}
```

- **المدخل:** `lessonData.voiceoverText` (النص العربي من JSON)
- **المخرج 1:** `public/voiceovers/lesson-02.mp3` (الملف الصوتي)
- **المخرج 2:** `public/timestamps/lesson-02.json` (توقيت كل كلمة)
- **التقنية:** edge-tts (Microsoft Edge TTS API المحلي)
- **الصوت الافتراضي:** `ar-EG-SalmaNeural` (صوت امرأة مصرية)

**ماذا لو فشل TTS؟** → يستخدم `fallback timing` (بدون ترجمة متزامنة)

#### الخطوة 3: عرض الفيديو عبر Remotion

```javascript
// run-factory.js - الأسطر 60-74
execSync(`npx remotion render LessonVideo "${rawVideoPath}" 
  --props="${dataPath}" --concurrency=4`, { stdio: "inherit" });
```

- **الأمر:** `npx remotion render` (عرض الفيديو)
- **المكون:** `LessonVideo` (المكون الرئيسي في `src/compositions/LessonVideo.tsx`)
- **المدخل:** مسار ملف JSON كـ `--props`
- **المخرج:** `public/lesson-02-raw.mp4` (فيديو خام غير مضغوط)
- **إعدادات الأداء:** `--concurrency=4` (4 أنوية فقط ← لا يثقل اللابتوب)

#### الخطوة 4: ضغط الفيديو

```javascript
// run-factory.js - الأسطر 77-90
execSync(`ffmpeg -i "${rawVideoPath}" -vcodec libx264 -crf 22 
  -preset fast -pix_fmt yuv420p -acodec aac -b:a 128k "${finalVideoPath}"`, 
  { stdio: "inherit" });
```

- **الأداة:** FFmpeg
- **برنامج الترميز (Codec):** H.264 (libx264) ← متوافق مع كل المتصفحات
- **الجودة:** CRF 22 (توازن ممتاز بين الجودة والحجم)
- **السرعة:** preset fast

## 8.3 generate_tts.py — توليد الصوت والترجمة المتزامنة

**المسار:** `video-factory/scripts/generate_tts.py`  
**الوظيفة:** تحويل النص العربي إلى صوت (MP3) مع توقيت كل كلمة (Word Timestamps)  
**اللغة:** Python 3.10+  
**عدد الأسطر:** 56 سطراً  
**المكتبة:** `edge-tts` (يجب تثبيتها: `pip install edge-tts`)

### الدوال (Functions)

#### `main()` (async)
- **الوصف:** النقطة الرئيسية - توليد الصوت والتوقيت بالتوازي
- **المعاملات (CLI Arguments):**

  | الوسم | النوع | إجباري | الوصف |
  |-------|-------|--------|-------|
  | `--text` | str | نعم | النص المراد تحويله إلى صوت |
  | `--voice` | str | لا | الصوت (افتراضي: `ar-EG-SalmaNeural`) |
  | `--output-audio` | str | نعم | مسار ملف MP3 الناتج |
  | `--output-timestamps` | str | نعم | مسار ملف JSON للتوقيت |

- **أمثلة الاستخدام:**
  ```bash
  python scripts/generate_tts.py \
    --text "أهلاً بيكم في درس جديد" \
    --voice ar-EG-SalmaNeural \
    --output-audio public/voiceovers/lesson-02.mp3 \
    --output-timestamps public/timestamps/lesson-02.json
  ```

- **الخوارزمية:**
  1. إنشاء كائن `edge_tts.Communicate` مع النص والصوت
  2. الاشتراك في حدث `WordBoundary` ← يسجل كل كلمة مع وقت بدايتها ومدتها
  3. كتابة الصوت إلى ملف MP3
  4. تحويل الـ HNS (hundred-nanosecond units) إلى ثوانٍ (÷ 10,000,000)
  5. حفظ التوقيت كـ JSON

### تنسيق مخرجات timestamps JSON

```json
[
  {
    "word": "أهلاً",
    "start": 0.12,
    "end": 0.45,
    "duration": 0.33
  },
  {
    "word": "بيكم",
    "start": 0.46,
    "end": 0.72,
    "duration": 0.26
  }
]
```

**كيف تستخدم في Remotion؟** ← `LessonVideo.tsx` (السطر 55-108) تستخدم هذه البيانات لعرض الترجمة المتزامنة (subtitles) أسفل الفيديو.

### الأصوات المتاحة للعربية

| الصوت | النوع | الوصف |
|-------|-------|-------|
| `ar-EG-SalmaNeural` | أنثى | مصرية - الموصى بها |
| `ar-SA-ZariyahNeural` | أنثى | سعودية |
| `ar-AE-FatimaNeural` | أنثى | إماراتية |
| `ar-EG-ShakirNeural` | ذكر | مصري |

### تثبيت edge-tts

```bash
pip install edge-tts
```

**لماذا edge-tts وليس غيره؟** لأنه:
- يعمل محلياً بالكامل (لا يحتاج API keys)
- مجاني (يستخدم Microsoft Edge TTS)
- يدعم العربية بطلاقة
- يعطي توقيت الكلمات (Word Boundary) مباشرةً

## 8.6 سير العمل الكامل: من PDF إلى فيديو

هذا هو السير الكامل للمشروع منذ البداية:

```
┌──────────────────────────────────────────────────────────┐
│    Content Extractor Pipeline (~2-3 ساعات)              │
│                                                          │
│  PDF → (pdf-to-images.py) → صور PNG                     │
│  صور → (extract-page.py) → JSON لكل صفحة (مع تبريد GPU)│
│  JSON → (merge-pages.py) → دروس JSON مدمجة              │
│  دروس → (generate-markdown.py) → JSON + Markdown        │
└──────────────────────────────────────────────────────────┘
                           ↓
              video-factory/src/data/lesson-XX.json
                           ↓
┌──────────────────────────────────────────────────────────┐
│    Video Factory Pipeline (~2-5 دقائق لكل درس)          │
│                                                          │
│  JSON → run-factory.js →                                 │
│    ① تحميل البيانات من JSON                             │
│    ② توليد الصوت (generate_tts.py → MP3 + Timestamps)   │
│    ③ عرض الفيديو (npx remotion render → MP4 خام)        │
│    ④ ضغط الفيديو (FFmpeg → H.264 MP4)                  │
│    ⑤ تنظيف الملفات المؤقتة                               │
└──────────────────────────────────────────────────────────┘
                           ↓
              video-factory/public/lesson-XX.mp4
                           ↓
              ✅ فيديو تعليمي جاهز للنشر!
```

## 8.7 استكشاف أخطاء إنتاج الفيديو

### "Data file not found"

**السبب:** ملف JSON غير موجود في `src/data/`

**الحل:**
```bash
# تأكد من تشغيل المرحلة 4 من content-extractor
cd content-extractor
python scripts/generate-markdown.py --input "merged-lessons/" --output "../src/data/"

# أو تأكد من اسم الملف
dir video-factory\src\data\lesson-02.json
```

### "Python TTS generation failed"

**السبب:** edge-tts غير مثبت

**الحل:**
```bash
pip install edge-tts
```

### "Remotion render failed"

**الأسباب المحتملة:**
- Node.js قديم جداً
- Remotion مثبت بشكل غير صحيح
- ملف JSON لا يحتوي على الحقول المطلوبة

**الحل:**
```bash
# تحقق من إصدار Node.js
node --version  # يجب أن ≥ 18

# أعد تثبيت Remotion
npm install

# تحقق من ملف JSON
cat src/data/lesson-02.json  # يجب أن يحتوي على voiceoverText
```

### "FFmpeg compression failed"

**السبب:** FFmpeg غير مثبت

**الحل:**
```bash
# ثبت FFmpeg من: https://ffmpeg.org/download.html
# أو استخدم winget
winget install FFmpeg

# بدون FFmpeg، الفيديو الخام سيبقى كبديل
```

### الفيديو النهائي كبير جداً

**السبب:** إعدادات CRF منخفضة

**الحل:** عدل `run-factory.js` السطر 84:
```javascript
// استخدم CRF أعلى = حجم أصغر (جودة أقل قليلاً)
-crf 28  // بدلاً من 22
```
