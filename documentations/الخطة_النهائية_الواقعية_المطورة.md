## 🎯 الرؤية المحدثة (اللي بنعمله ولمين)

**المنتج:** منصة تعليمية تفاعلية للثانوية العامة المصرية (15-18 سنة)  
**الجمهور:** طلاب الثانوية العامة في مصر - شعبة علمي (رياضة + علوم) + أدبي  
**التمايز:**
- محاكيات تفاعلية (HTML5 Canvas) - مفيش منصة مصرية بتعملها كويس
- فيديوهات تعليمية مولدة ذاتياً من أفضل الكتب الخارجية (المعاصر + الامتحان + التفوق) مدمجة مع كتب الوزارة
- نظام اختبارات ذكي مع Feedback فوري وصحيح
- خرائط ذهنية + انفوجرافيك لكل درس

**نموذج العمل:** Freemium (محتوى مجاني + اشتراك مدفوع 99-199 جنيه/شهر)  
**المدة:** 18 شهر (3 مراحل)  
**الفريق:** أنت (Product Owner + Content Supervisor) + 3 AI Agents + مبرمج واحد (Full Stack)  
**الميزانية:** ~3000-5000 دولار + 2000-4000 جنيه مصري (للـ MVP)

## 🤖 نظام الـ 3 AI Agents بالتوازي

### لماذا 3 Agents؟

محاولة جعل AI واحد يعمل كل شيء = فشل. كل Agent له تخصص واحد وواضح.

```
┌─────────────────────────────────────────────────────────────┐
│  Agent 1: "المستخرج" (Extractor)                            │
│  الأدوات: Ollama + Qwen2-VL:7b (محلي)                     │
│  المهمة: قراءة صفحات الكتب (صور) → استخراج المحتوى الخام    │
│  المدخلات: صور PNG (صفحات الكتب)                          │
│  المخرجات: ملفات JSON خام (Raw)                           │
│  السرعة: 30 ثانية/صفحة (مع Cooldown)                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Agent 2: "المدقق" (Validator)                              │
│  الأدوات: Ollama + Gemma3:4b (محلي) + Python Scripts      │
│  المهمة: Cross-Validation + Automated Formula Check       │
│  المدخلات: JSON خام + صورة الصفحة الأصلية                 │
│  المخرجات: JSON مصحح + تقرير ثقة (Confidence Score)      │
│  السرعة: 20 ثانية/صفحة                                    │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Agent 3: "الباني" (Builder)                                │
│  الأدوات: Google AI Studio (Gemini) + Z.ai (Zed)          │
│  المهمة: تحويل JSON → Markdown نهائي (6 Chunks) + كود      │
│  المدخلات: JSON مصحح من Agent 2                           │
│  المخرجات: Markdown + سكريبت فيديو + كود المنصة            │
│  السرعة: 5 دقائق/درس (Markdown) + 1-3 أيام/Module (كود)    │
└─────────────────────────────────────────────────────────────┘
```

### قواعد التوازي

1. **Agent 1 و Agent 2 يشتغلوا بالتوازي:** لما Agent 1 يخلص صفحة 1 ويحفظها، Agent 2 يبدأ يدقق صفحة 1 في نفس الوقت اللي Agent 1 بيشتغل فيه على صفحة 2
2. **Agent 3 ما يبدأش إلا لما يكون عنده 7 دروس (فصل كامل) مصححة:** مش هنبني كود على محتوى ناقص
3. **الإنسان (أنت) يتدخل في 3 نقاط فقط:**
   - مراجعة العينة (10% من الصفحات المشكوك فيها) - 2-3 ساعات
   - اختبار المحاكي على الموبايل - 1 ساعة
   - قراءة الـ Markdown النهائي قبل رفعه - 30 دقيقة/درس

## 📚 مصادر المحتوى (الحقيقية - تم التأكد منها)

### الكتب الخارجية (الأفضل تنظيماً - تم جمعها من Telegram + مواقع)

| المصدر | النوع | ليه أحسن؟ | الاستخدام |
|--------|-------|-----------|-----------|
| **المعاصر** | شرح + تدريبات + أمثلة | شرح مبسط + عامية + تدريبات كتيرة | المرجع الرئيسي للشرح |
| **الامتحان** | شرح + أسئلة امتحانية | "أرشادات" للحل + أسئلة متوقعة | بنك الأسئلة + أرشادات |
| **التفوق** | مراجعة + ملخصات | ملخصات نهائية + خرائط ذهنية | المراجعة + الخرائط |
| **الأضواء** | شرح مفصل | شرح عميق للمفاهيم | المفاهيم الصعبة |

### الكتب الحكومية (للتأكد فقط)

| المصدر | الرابط | الاستخدام |
|--------|--------|-----------|
| بنك المعرفة المصري | `ekb.eg` | التأكد من المنهج الرسمي + الصيغ الدقيقة |
| منصة ذاكر | `study.ekb.eg` | مراجعات إضافية |
| موقع الوزارة | `moe.gov.eg` | توزيعات المنهج |

### ⚠️ القاعدة القانونية

**لا ننسخ من أي كتاب.** نستخدم الكتب الخارجية كـ **مرجع للأفكار** فقط:
- المعاصر شرح قانون أوم بـ 3 خطوات → نكتب شرحنا بـ 3 خطوات مختلفة
- الامتحان حط مثال بطارية 9V → نحط مثال شاحن الموبايل
- التفوق رسم خريطة ذهنية → نرسم خريطتنا بشكل مختلف
- **الصيغ الرياضية = من كتاب الوزارة (بنك المعرفة) = public domain**

## 🔄 سير عمل المحتوى (Content Pipeline) - مطور بالكامل

### المرحلة 1: استخراج المحتوى (أسبوع 1)

**الهدف:** 7 دروس (الفصل الأول: التيار الكهربي) من كتاب المعاصر + التفوق + الامتحان

#### اليوم 1: تحضير الكتب

1. **جمع الكتب (2-3 ساعات):**
   - افتح Telegram → ادخل `t.me/Thanwya_PDF` أو `t.me/kotob_3ama`
   - حمل: `المعاصر فيزياء 3 ثانوي.pdf` + `الامتحان فيزياء 3 ثانوي.pdf` + `التفوق فيزياء 3 ثانوي.pdf`
   - حمل كتاب الوزارة من `ekb.eg` (تسجيل بالرقم القومي)
   - حطهم في `content-extractor/books/`

2. **تحويل الفصل الأول لصور (30 دقيقة):**
   - شغل `01_pdf_to_images.py`
   - المدخل: `books/moaser-physics-3.pdf` (صفحات 15-45 مثلاً = الفصل الأول)
   - المخرج: `pages/page_015.png` إلى `pages/page_045.png`
   - **إعدادات ثابتة:** DPI 150، Max Size 512px، صيغة PNG

#### اليوم 2-3: Agent 1 (المستخرج) يشتغل

**السكريبت `02_extract_page.py` يشتغل:**

```
لكل صفحة في pages/:
    1. يرسل صفحة واحدة بس لـ Qwen2-VL:7b (Ollama)
    2. ينتظر 10 ثواني (Cooldown)
    3. يحفظ النتيجة فوراً في raw-json/page_XXX.json
    4. يطبع: "صفحة 15/30 - خلصت - نوع: lesson"
    5. لو قفل الجهاز: يشغل تاني، يكمل من آخر صفحة (Resume)
```

**الـ Prompt اللي يرسله Agent 1 للموديل:**

> "أنت قارئ كتب مدرسية مصري متخصص. اقرأ هذه الصفحة من كتاب الفيزياء للثانوية العامة واستخرج المعلومات في JSON صحيح فقط.
>
> المطلوب:
> 1. `page_type`: "lesson" أو "exercise" أو "formula" أو "empty"
> 2. `lesson_title`: عنوان الدرس (لو موجود)
> 3. `content_summary`: ملخص بسيط لمحتوى الصفحة
> 4. `formulas`: ["$I = \frac{Q}{t}$", "$V = IR$"] (LaTeX بين $)
> 5. `definitions`: [{"term": "شدة التيار", "definition": "..."}]
> 6. `examples`: [{"question": "...", "solution": "..."}]
> 7. `exercises`: ["سؤال 1", "سؤال 2"]
> 8. `figures_description`: وصف الرسومات
> 9. `tables_description`: وصف الجداول
> 10. `source_hints`: أفكار من الكتاب (مش نص حرفي)
>
> قواعد:
> - JSON صحيح فقط. لا نص خارج JSON.
> - الصيغ: LaTeX بين $ $
> - لا تخترع معلومات. لو الصفحة فارغة: `{"page_type": "empty"}`
> - اكتب بالعربية الفصحى (مش عامية)."

**الوقت المتوقع:** 30 صفحة × 30 ثانية = 15 دقيقة تشغيل فعلي + Cooldown = ~45 دقيقة

#### اليوم 3-4: Agent 2 (المدقق) يشتغل بالتوازي

**السكريبت `03_cross_validate.py`:**

```
لكل ملف في raw-json/:
    1. يقرأ JSON من Agent 1
    2. يرسل نفس الصفحة (الصورة) لـ Gemma3:4b (موديل مختلف)
    3. يقارن النتيجتين:
       - لو الصيغ متطابقة 100% → Confidence: HIGH
       - لو اختلفوا في رقم → Confidence: MEDIUM → علامة "⚠️ رقم مختلف"
       - لو اختلفوا في صيغة → Confidence: LOW → علامة "⚠️ صيغة مختلفة"
    4. يحفظ في validated-json/page_XXX.json
```

**السكريبت `04_auto_validate.py` (Python - لا يحتاج AI):**

```
لكل ملف في validated-json/:
    1. يفحص LaTeX: هل `$...$` مكتوبة صح؟
    2. يحسب الأمثلة الحسابية: Q=30, t=5 → I=6A. لو الموديل كتب 5A = غلط
    3. يفحص الوحدات: A, V, Ω, C, s
    4. يفحص التناقضات: صفحة 5: I=Q/t، صفحة 6: I=Q×t = تناقض
    5. يولد تقرير: validation-report.json
```

**الوقت المتوقع:** 30 صفحة × 20 ثانية = 10 دقائق + 5 دقائق للفحص الآلي = 15 دقيقة

#### اليوم 5: المراجعة البشرية (أنت - 2-3 ساعات فقط)

**لا تفحص كل حاجة. افحص العينة فقط:**

1. افتح `validation-report.json`
2. دور على الصفحات اللي عليها علامة "⚠️" (Confidence: LOW أو MEDIUM)
3. **افحص الحتة دي بس:**
   - مثلاً: 5 صفحات من 30 (مش 30 صفحة)
   - 2-3 دروس من 7 (مش 7 دروس)
4. قارن الصيغ المختلفة بكتاب الوزارة (المرجع الرسمي)
5. صلح في `validated-json/` يدوياً (مش لازم تكتب الكل - عدل الرقم الغلط بس)

**الوقت:** 2-3 ساعات (مش سنة)

#### اليوم 6-7: الدمج + Markdown

**السكريبت `05_merge_pages.py`:**

```
المدخل: validated-json/ + book-index.json (أنت هتكتبه يدوياً - 10 دقائق)

book-index.json:
{
  "lessons": [
    {"id": "phy-3-1-1-01", "title": "مقدمة في الكميات الفيزيائية", "pages": [15, 16, 17]},
    {"id": "phy-3-1-1-02", "title": "شدة التيار الكهربي", "pages": [18, 19, 20, 21]},
    ...
  ]
}

السكريبت يدمج JSON الصفحات (15+16+17) → درس واحد JSON
```

**السكريبت `06_generate_markdown.py`:**

```
المدخل: merged-lessons/lesson-01.json
المخرج: final-markdown/lesson-01.md (الـ 6 Chunks)

الـ 6 Chunks:
1. 🎬 Chunk 1: سكريبت الفيديو (10-15 دقيقة، بالعامية المصرية)
2. 🧠 Chunk 2: الخريطة الذهنية (JSON للعقد والفروع)
3. 📖 Chunk 3: الشرح المقروء + الانفوجرافيك (Markdown + HTML)
4. 🧪 Chunk 4: وصف المحاكي (JSON للـ Simulator Config)
5. ⚡ Chunk 5: بنك الأسئلة (5 أسئلة: 3 MCQ + 1 Numerical + 1 Conceptual)
6. 🏆 Chunk 6: بطاقات المراجعة (Flashcards YAML)
```

**الوقت:** 7 دروس × 5 دقائق = 35 دقيقة تشغيل

**الناتج بعد أسبوع 1:** 7 ملفات Markdown جاهزة في `final-markdown/`

## 🎬 سير عمل الفيديوهات (Video Pipeline) - مطور

### المرحلة 2: مصنع الفيديوهات (أسبوع 2)

**الهدف:** 7 فيديوهات (دقيقة وربع لكل درس) = ملخص الدرس

#### الفكرة المطورة: الفيديو مش بديل الدرس - الفيديو "بوستر" للدرس

| النوع | المدة | المحتوى | الغرض |
|-------|-------|---------|-------|
| **فيديو الملخص** | 75 ثانية (دقيقة وربع) | أهم 3 أفكار + صيغة واحدة + محاكي سريع | يجذب الطالب يدخل الدرس |
| **الدرس الكامل** | 10-15 دقيقة | شرح مفصل + أمثلة + تدريبات | جوه المنصة (مش فيديو واحد) |
| **المراجعة** | 5 دقائق | ملخص + خريطة ذهنية + أسئلة | قبل الامتحان |

**الفيديو 75 ثانية = 7 مشاهد (Scenes):**

| الثانية | المشهد | اللي بيظهر | الصوت بيقول |
|---------|--------|------------|-------------|
| 0-4 | Intro | شعار المنصة + موسيقى خفيفة | - |
| 4-12 | العنوان | عنوان الدرس + اسم الفصل | "درسنا النهاردة: شدة التيار الكهربي" |
| 12-24 | الفكرة 1 | الصيغة تتكتب حرف حرف (Typewriter) | "التيار = الشحنة ÷ الزمن... I = Q على t" |
| 24-40 | المحاكي | Slider يتحرك + أميتر يتغير | "دلوقتي جرب بنفسك... غير الجهد وشوف" |
| 40-55 | الخريطة | عقد الخريطة الذهنية تتكون | "نلخص: التيار → تعريف → قياس → تطبيقات" |
| 55-68 | الاختبار | سؤال + إجابات (تظهر واحدة واحدة) | "اختبر نفسك: إذا V=10 و R=5... الإجابة؟" |
| 68-75 | Outro | شعار + "ادخل الدرس الكامل على المنصة" | - |

#### سير العمل (يوم بيوم)

**اليوم 1: توليد الصوت (TTS)**
- الأداة: Edge-TTS (مجاني - صوت Salma المصرية)
- المدخل: Chunk 1 (سكريبت الفيديو) من Markdown
- المخرج: `voiceovers/lesson-01.mp3` + `voiceovers/lesson-01-timestamps.json` (توقيت كل كلمة)
- الوقت: 7 دروس × 2 دقيقة = 14 دقيقة

**اليوم 2-3: بناء المشاهد (Remotion)**
- Agent 3 (Gemini/Z.ai) يكتب Remotion Components:
  - `SceneIntro.tsx`, `SceneTitle.tsx`, `SceneFormula.tsx`, `SceneSimulator.tsx`, `SceneMindMap.tsx`, `SceneQuiz.tsx`, `SceneOutro.tsx`
- كل Component بيستقبل `props` (بيانات الدرس) ويرender animation
- الوقت: 3 أيام (للـ 7 Components)

**اليوم 4: Render الفيديوهات**
- `npx remotion render LessonVideo lesson-01.mp4`
- اللابتوب بيفتح Chrome Headless ويسجل frame by frame
- جودة: 1080p, 30fps
- وقت الـ Render: 2-5 دقائق/فيديو (حسب CPU)
- 7 فيديوهات = 20-35 دقيقة تشغيل

**اليوم 5: ضغط + رفع**
- FFmpeg يضغط: `ffmpeg -i lesson-01.mp4 -vcodec h264 -crf 22 lesson-01-final.mp4`
- الحجم النهائي: 5-15 MB/فيديو
- رفع على Cloudflare R2 (أو أي CDN)
- حفظ الرابط في قاعدة البيانات

   - Next.js Server Actions بس

   - لا Microservices

   - لا تخمين في العلم

   - شغل على اللي مكتوب في الخطة دي بس

   - "Dashboard للمعلمين" = مشروع منفصل (للمرحلة المتقدمة)
